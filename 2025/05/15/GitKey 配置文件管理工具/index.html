GitKey 配置文件管理工具

GitKey 配置文件管理工具

项目简介

本项目是一个用于管理多套 Git SSH 密钥和用户信息的命令行工具，支持一键切换、生成、删除、测试 SSH 密钥，并可记录操作日志。适用于需要频繁切换 Git 账号或多平台（如 GitHub、GitLab）开发的用户。

主要功能

  • 密钥条目管理：支持添加、删除、修改、查看所有密钥条目（Index、UserName、Email、Remark）。
  • SSH 密钥管理：为指定条目生成/删除/查看 SSH 密钥，支持多种算法（RSA、ED25519、DSA、ECDSA）。
  • Git 配置切换：一键切换当前全局 Git 用户名、邮箱，并自动复制对应 SSH 密钥到系统 .ssh 目录。
  • SSH 连接测试：可选择测试 GitHub 或 GitLab 的 SSH 连接，自动识别连接结果。
  • 日志记录：所有关键操作均会写入 logs 目录下的按日期日志文件。
  • 备注管理：每个密钥条目支持备注（Remark），便于区分不同用途。

2. 主菜单功能

  • 1. 查看条目：以表格形式显示所有密钥条目及其备注、密钥状态。
  • 2. 添加条目：先显示全部条目，再交互式添加新条目（支持备注），可选择是否生成 SSH 密钥。
  • 3. 删除条目：支持二次确认，删除条目及其对应密钥文件夹。
  • 4. 修改条目：支持用户名、邮箱、备注的修改。
  • 5. SSHKey管理：对指定条目进行密钥生成、删除、查看等操作。
  • 6. 当前Git配置：显示当前全局 Git 用户名和邮箱。
  • 7. 切换Git配置：选择条目后自动切换全局 Git 配置并复制密钥到 .ssh 目录。
  • 8. 测试SSH连接：可选 GitHub 或 GitLab，自动识别连接结果。
  • 0. 退出脚本

3. 其它说明

  • 配置文件：gitkey.json 自动维护，无需手动编辑。
  • 日志文件：所有操作日志保存在 logs/ 目录，便于追溯。
  • 密钥目录：每个条目的密钥文件存放于 SSHKey/Index/ 目录下。
  • 备注字段：可自定义填写，如“GitHub”、“GitLab”、“公司账号”等。

适用环境

  • Windows 系统（推荐使用 main.bat 启动）
  • 需预装 Python 3.x
  • 需已安装 ssh-keygen 工具（Git Bash 或 Windows OpenSSH）
